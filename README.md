## 🏗️ 系统架构

本项目是一个典型的闭环控制系统。为了实现更高效的温控，我们在执行端设计了水循环冷却链路，以提高 TEC 的热交换效率。

![系统架构图](./images/系统架构图)

## 🔌 硬件选型与设计

在硬件选型上，充分考虑了系统的实时性、精度以及大惯性水体热力学特性。

- **主控芯片**：STM32F103C8T6。
- **执行机构**：TEC1-12704 半导体制冷片（支持双向温控）。
- **执行机构驱动**：BTS7960 大电流 H 桥电机驱动模块。
- **水循环系统**：12V 微型水泵 + 水冷头，加速热交换。
- **温度采集**：K型热电偶 + MAX31855 模块，提供宽范围高精度反馈。
- **用户交互**：旋转编码器（设定目标温度） + OLED 屏幕（状态显示）。
- **电源管理**：12V 5A 开关电源，搭配 12V 转 5V 降压芯片为控制逻辑供电。

## ⚙️ 软件架构与 FreeRTOS 任务

基于 CLion 开发环境，采用 FreeRTOS 框架保证系统硬实时性，将系统功能划分为三个任务：

| 任务名称 | 优先级 | 备注 | 描述 |
| :--- | :--- | :--- | :--- |
| `vTaskControlLoop` | High (2) | 核心控制 | 运行复合 PID 计算、温度采集、PWM 更新。 |
| `vTaskUIHandler` | Medium (1) | 异步响应 | 处理旋转编码器输入，更新目标温度设定。 |
| `vTaskDisplay` | Low (0) | 刷新率控制 | OLED 屏幕显示目标温度、实际温度及系统状态。 |

## 🔥 核心算法实现详解

### 高级复合 PID 控制算法

针对水体的大滞后和大惯性特性，引入复合 PID 控制算法以消除积分饱和、提高响应速度和阻尼：

1.  **积分分离**：当目标温度与实际温度偏差大于门限 $\Delta T$ 时，仅执行 $P$ 或 $D$ 控制；当 $\Delta E \leq \Delta T$ 时，引入 $I$ 项。大幅抑制全功率运行造成的严重超调。
2.  **积分抗饱和**：对积分项设置固定的输出限幅 $[I_{min}, I_{max}]$。防止 TEC 全功率工作时积分过度累积。
3.  **微分先行**：将 PID 微分项计算由误差微分 $\frac{de}{dt}$ 改为传感器反馈量微分 $\frac{d(PV)}{dt}$。避免目标设定值突变时产生过大输入脉冲。
4.  **低通滤波**：对原始温度数据和微分项进行低通滤波，抑制高频杂波，平滑控制量。
